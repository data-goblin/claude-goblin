# Version 1.1.1 - Windows Hardening

Released: July 14, 2026

## Overview

Follow-up to 1.1.0 from a full Windows-compatibility pass over the codebase.

## Fixes

### Hook commands now work under cmd.exe

`ccg setup hooks usage` and `ccg setup hooks png` wrote POSIX shell syntax
into settings.json:

```
ccg update usage > /dev/null 2>&1 &
```

On Windows there is no `/dev/null` and `&` is a command separator, so the
hooks failed on every fire. On Windows they now write:

```
ccg update usage >NUL 2>&1
```

The png export path is also quoted, fixing home directories containing
spaces on every platform.

### UTF-8 file IO everywhere

settings.json, goblin_config.json, bundled skill markdown, and generated
hook/devcontainer files were opened with the locale default encoding, which
is cp1252 on Windows. Reading UTF-8 content written by Claude Code (for
example paths containing non-ASCII usernames) could crash or corrupt data.
All user-file reads and writes now pass `encoding="utf-8"` explicitly.

### awesome-hooks guarded on Windows

The awesome-hooks (`bundler-standard`, `file-name-consistency`,
`uv-standard`) are shebang scripts registered by bare path, which Windows
cannot execute. Their setup now exits with a clear message on Windows
instead of installing hooks that fail on every tool call.

## Removed

Dead `visualization/activity_graph.py` module: imported nowhere and broken
at import time by the same `datetime.date` annotation shadowing fixed in
1.1.0.

## Audit notes

Also verified clean during the pass: no POSIX-only module imports remain,
no Windows-missing `os`/`signal` APIs, no POSIX-only `strftime` codes,
subprocess callers resolve `az`/`fab` via `shutil.which` (so `.cmd` shims
work), JSONL/Codex parsers already read UTF-8, and all 65 modules import
with `pty`/`termios`/`tty`/`fcntl`/`grp`/`pwd` blocked.

# Version 1.2.0 - Hermes Agent Usage

Released: July 23, 2026

## Overview

Hermes Agent model calls can now be ingested into the same `usage_records`
table as Claude Code and Codex. This keeps existing token aggregations and
dashboards query-compatible while adding persistent agents that do not have a
natural Stop-hook lifecycle.

Configure the directory containing content-free Hermes usage events as an
extra source:

```json
{
  "path": "/var/lib/bonk-usage",
  "device_id": "bonk-ouroboros1",
  "device_name": "ouroboros1",
  "device_type": "linux",
  "format": "hermes"
}
```

Each JSONL event represents one successful API response and is mapped onto the
existing schema. The event identity becomes `message_uuid`, Hermes' session ID
becomes `session_id`, cache writes map to `cache_creation_tokens`, and
`version` records `hermes:<provider>`.

The producer must not include prompts, responses, reasoning text, tool
arguments, user identifiers, or credentials. CCG only needs timestamps,
identities, model/provider labels, platform/cwd metadata, and numeric usage
buckets.

# Version 1.2.1 - Generic Hermes Usage Integration

Released: July 23, 2026

## Overview

Hermes Agent usage ingestion is model- and provider-agnostic. The exporter
consumes Hermes' normalized `post_api_request` usage fields, so the same
pipeline works with local inference servers, hosted APIs, routed models, and
future providers without per-model parsing.

Deployments choose their own CCG account, Hermes account, service name, spool
directory, device ID, and device name. CCG continues to map every request into
the existing `usage_records` schema.

A representative extra source is:

```json
{
  "path": "/var/lib/ccg/hermes-usage",
  "device_id": "hermes-agent-host",
  "device_name": "Hermes Agent (host)",
  "device_type": "linux",
  "format": "hermes"
}
```

# Version 0.1.5 - Stats Fast Mode

Released: October 13, 2025

## Overview

This release adds the `--fast` flag to the `stats` command and fixes missing limits tracking.

## New Features

### Fast Mode for Stats

New `--fast` flag skips all database updates for faster rendering:

```bash
ccg stats --fast
```

Reads directly from the existing database without:
- Parsing JSONL files
- Updating token records
- Capturing current limits

Useful when you just want to view stats quickly without waiting for updates.

## Bug Fixes

### Missing Limits in Stats

**Problem:** The `stats` command wasn't saving limits to the database like other commands (`usage`, `export`), causing limits data to be incomplete.

**Solution:** Added automatic limits capture and save to the `stats` command:

```python
# Now matches behavior of usage and export commands
limits = capture_limits()
if limits and "error" not in limits:
    save_limits_snapshot(...)
```

## Technical Changes

- Added `--fast` flag to `stats` command matching `usage` and `export`
- Added `capture_limits()` call to `stats` command
- Added `save_limits_snapshot()` call to persist limits data
